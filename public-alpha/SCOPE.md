# public alpha scope

## Included

- draft SKILL documents (`core`, `auth`, `filesystem`, `orm`)
- validated SAR anti-pattern references
- high-level summaries of Phase α/β/γ findings

## Not included as public claims

- universal claims about all languages or all domains
- production-readiness guarantees
- broad claims that "3 axes" define code quality
- MCP/tool-combination claims as validated results

## Raw experiment data policy

Raw experiment files remain in the repository for transparency, but they should be interpreted as:

- reproducibility artifacts
- internal research evidence
- not polished end-user documentation

## Safe public statement

The safest public statement is:

> In limited experiments, structured SKILL prompts with explicit decision criteria and autonomous follow-up improved LLM analysis quality in some domains, and domain-specific drafts sometimes outperformed a general fallback prompt.

## Unsafe public statements

Do **not** say:

- "these skills are production-ready"
- "this framework solves code quality"
- "the 3 axes are the true decomposition of software quality"
- "all models benefit equally"
