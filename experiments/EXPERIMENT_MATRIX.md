# 실험 목록표 (Experiment Matrix)

## 연구 질문 (순서대로)

1. **SKILL(가이드)이 LLM 분석 품질을 높이는가?** ← 현재 단계
2. 효과가 있다면, 어떤 구체성 수준(간략/체크리스트/상세)에서 나는가?
3. 최소 토큰으로 최대 효과를 내는 가이드 구조는 무엇인가? (축 균형, 압축)
4. MCP(정량 도구)와 결합하면 추가 효과가 있는가?

> 1번이 실패하면 2~4는 진행하지 않는다.
> 🍞🧀🥓는 가이드의 예시이지, "코드 품질이 3축이다"라는 주장이 아니다.
> 보안(🍞) 축을 첫 번째 사례로 실험한다.
## 실험 전 체크리스트 (매 실험 단위 전 확인)

```
□ 독립변인이 정확히 1개만 변하는가? → 가이드 구체성 수준 (A/B/C)
□ 나머지 변인이 모두 동일한가? → 코드, 모델, temperature, system prompt
□ 주석/힌트가 답을 알려주지 않는가? → 코드 원본 그대로
□ 실제 오픈소스 코드인가? → MIT/BSD, 커밋 해시 고정
□ 제3자가 재현 가능한가? → run_experiment.py, .env, SHA-256, 모델 ID
□ 이론 범위를 초과하여 주장하지 않는가? → 채점 기준 사전 정의
□ 이전 교란 변인이 반복되지 않는가? → v1(지시 차이), v2(모델 미통제) 해소
```

## Phase α 중간 발견 (bread-R1, 2026-03-18)

**SKILL이 선별 조건(체크리스트)만 제시하면 LLM은 task 완료로 판정하고 탐색을 종료한다.**
이것은 출력 토큰 한계와 무관하다 (max_tokens를 2배로 늘려도 동일).
"체크리스트 완료 후 자율 탐색" 지시를 추가하면 시야 축소가 해소된다.

→ 실험 그룹에 D(SKILL + 자율 탐색)를 추가.

## 측정 지표 (Phase α 데이터에서 도출)

> 사전 정의가 아닌, Phase α에서 실제로 B vs D 간 차이가 나타난 항목만 지표로 채택.
> 차이가 없었던 것(CC 수치 정확도, CC 방향성)은 SKILL 효과 측정에 부적합하므로 제외.

| 지표 | 정의 | 측정 방법 | α에서 차이 |
|------|------|----------|-----------|
| **탐지력** | 실제 이슈를 몇 개 찾는가 | TP 수 (3채점자 2/3 합의) | ✅ D > B +33% |
| **정밀도** | 찾은 것 중 실제 이슈 비율 | TP / (TP + FP) | ⚠️ 미미 (β에서 재확인) |
| **가이드 효과** | SKILL 규칙이 유도한 탐지 수 | D에서만 찾고 B에서 못 찾은 TP | ✅ 3레포에서 관찰 |
| **자율 탐색 효과** | 체크리스트 밖 추가 탐지 수 | D의 자율 탐색 구간 TP | ✅ 시야 축소 해소 |
| **구체성** | 위치 지정 정밀도 | 0: 일반 / 1: 함수명 / 2: 줄+코드 | ✅ D ≥ B |
| **토큰 효율** | 이슈당 토큰 비용 | 출력 토큰 / TP | ⚠️ α에서 방향만 관찰 |

### TP/FP 판정 기준

| 판정 | 기준 |
|------|------|
| **TP** | 위험이 존재하고, 어떤 수준에서든 완화/해결이 가능 |
| **FP** | 위험이 존재하지 않음 |

"이 코드에서 직접 고칠 수 없다"는 FP의 근거가 아니다.

## Phase β 결론 및 γ 전환 결정 (2026-03-18)

### β 결과 요약

| 지표 | 값 | β 성공 기준 | 충족? |
|------|---|------------|------|
| D > B (TP 수) | 4/12 (33%) | 8/12 이상 | ❌ |
| D 정밀도 > B 정밀도 | 8/12 (67%) | — | — |
| D-only TP 발견 | 7/12 | 3개 이상 | ✅ |
| B > D | 2/12 | 0 | ❌ |

### γ 진행 결정 근거

TP 수 기준 β 성공 기준 미달 (4/12). 진행 규칙상 "β p > 0.3이면 가설 폐기 검토" 구간.
그러나 다음 근거로 가설 폐기 대신 γ 진행을 결정:

1. **정밀도 우위 (8/12)** — TP 수가 같아도 D가 FP를 덜 만든다. SKILL의 효과가 "더 많이 찾는"에서 "더 정밀하게 찾는"으로 재정의 필요.
2. **D-only TP (7/12)** — D가 B에서 놓치는 특정 이슈를 찾는 능력은 유지됨. SKILL이 도메인 특화 이슈에 유효.
3. **코드 유형 의존성 발견** — α(보안 인증 코드)에서 강했던 효과가 β(다양한 코드)에서 약화. 이는 "SKILL 효과 없음"이 아니라 "코드 유형별 차이 존재"의 신호.
4. **γ 목적 전환** — B vs D 비교에서 → **D-general vs D-specific** (범용 SKILL vs 도메인 특화 SKILL) 비교로 실험 질문 변경. β 결과가 이 질문을 촉발.

> 이 결정은 가설 폐기 기준(p > 0.3)을 기계적으로 적용하지 않고,
> 데이터에서 새로운 신호(정밀도 우위, 코드 유형 의존성)를 읽어 실험 질문을 정제한 것.
> RESEARCH.md의 "단계 간 적응적 조정" 원칙 적용.

## Phase β — α 결과 기반 확장

> α에서 확인: D(SKILL+자율) > B(체크리스트), 3축 재현, 시야 제어 효과
> β 목적: 새로운 축과 다양한 모듈에서 동일 패턴이 재현되는가 + SKILL 내용 축적

### β 설계 원칙
- **같은 레포를 반복하지 않음** — 새로운 모듈마다 새로운 SKILL 규칙이 도출되어야 함
- **B vs D만 비교** — α에서 A/C는 역할을 다 함 (A≈B, C<D 확인됨)
- **실험 모델**: gpt-5.4, sonnet-4.6 — gpt-5.3-codex는 계획에만 있었고 미사용. 실제 β/γ 실험은 gpt-5.4(풀 모델)로 실행됨 (확인 2026-03-26)
- **채점**: Opus 4.6 주 채점, 불일치 시 GPT-5.4 교차 확인
- **각 실험이 새로운 SKILL 규칙을 테스트** — 동일 B1-B4를 반복하지 않음

### 실험 단위

12 실험 × 2 모델 × 2 그룹(B, D) = **48 단위**

### β에서 도출될 것

각 실험에서 **D가 B보다 추가로 찾은 이슈**를 수집한다. 이것들이 SKILL 규칙의 후보가 된다.
사전에 축이나 규칙 ID를 정하지 않는다 — 데이터에서 자연스럽게 분류 체계가 나온다.

D가 B와 차이 없는 실험의 규칙 후보는 폐기.

### β 성공 기준

- 12개 실험 중 **8개 이상에서 D > B** (TP 또는 구체성)
- 새로운 규칙 후보 중 **3개 이상이 D에서만 탐지**되면 SKILL 규칙으로 확정
- 교차 축(β-11~12)에서 **시야 축소가 재발하는지** 확인 → 발생 시 축별 분리 실행 권고

## 진행 규칙

```
α 방향성 있음 → β
α 방향성 없음 → 프롬프트 재설계 후 α 재실행

β p < 0.1 → γ
β p > 0.3 → 가설 폐기 검토

γ p < 0.05 + 효과 크기 중간 + 3언어 → δ
γ 불충분 → 적응적 조정 후 γ 재실행

δ 완료 → Tier 2 승격/폐기 최종 판정
```

### 단계 간 적응적 조정
- 특정 언어/도메인 효과 집중 → 해당 축 모수 집중
- 효과 없는 축 → 최소 유지 (반증 근거)
- 예상 외 경향 → 즉시 원인 분석 후 설계 반영
- 모든 조정 사항은 이 문서에 diff로 기록
