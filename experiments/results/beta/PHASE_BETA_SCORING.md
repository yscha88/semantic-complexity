## 해석

1. **Phase α의 "D > B +33%" 패턴은 β에서 약화됨** — 모든 코드에서 D가 우월한 것이 아님
2. **D-only TP는 7/12에서 관찰** — D가 B에서 놓치는 특정 이슈를 찾는 능력은 존재
3. **D 정밀도가 8/12에서 B보다 높음** — D가 FP를 덜 만드는 경향
4. **TP 총 수에서는 D ≈ B** — Phase α의 극적 차이는 보안 인증 코드에 특화된 것일 수 있음
5. **Sonnet에서 D 효과가 더 약함** — 모델별 차이 존재

## 결론

SKILL(D)의 효과는 **코드 유형에 따라 달라진다**:
- 보안 인증 코드 (α): D >> B (강한 효과)
- 다양한 코드 (β): D ≥ B (약한 효과, 정밀도에서 우위)

**SKILL은 보편적으로 "더 많이 찾는" 도구가 아니라, "더 정밀하게 찾고, 특정 이슈를 놓치지 않게 하는" 도구.**
